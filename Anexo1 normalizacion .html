Anexo 5 – Descripción del Diagrama Entidad–Relación (3FN)

Este anexo describe el ERD final para tu caso práctico. .

  • Estudiante (id_estudiante, rut, nombre, nacionalidad)
  • Profesor (id_profesor, rut, nombre)
  • Curso (id_curso, nombre_curso, id_profesor)
  • Estudiante_Curso (id_estudiante, id_curso)

Relaciones clave:

  • Un Estudiante puede estar en muchos Curso (M:N) vía Estudiante_Curso.
  • Un Curso es dictado por un Profesor (1:N desde Profesor a Curso).
  • Un Profesor puede dictar varios Curso.

Sugerencia: añade cardinalidades (1, N) sobre las líneas del diagrama para mayor claridad.

Anexo 6 – SQLite Studio

Ejecuta el siguiente script en SQLite Studio para crear las tablas (3FN) y cargar los datos de ejemplo. 

-- Creación de tablas (3FN)
CREATE TABLE Estudiante (
    id_estudiante INTEGER PRIMARY KEY,
    rut TEXT NOT NULL,
    nombre TEXT NOT NULL,
    nacionalidad TEXT NOT NULL
);

CREATE TABLE Profesor (
    id_profesor INTEGER PRIMARY KEY,
    rut TEXT NOT NULL,
    nombre TEXT NOT NULL
);

CREATE TABLE Curso (
    id_curso INTEGER PRIMARY KEY,
    nombre_curso TEXT NOT NULL,
    id_profesor INTEGER,
    FOREIGN KEY (id_profesor) REFERENCES Profesor(id_profesor)
);

CREATE TABLE Estudiante_Curso (
    id_estudiante INTEGER,
    id_curso INTEGER,
    PRIMARY KEY (id_estudiante, id_curso),
    FOREIGN KEY (id_estudiante) REFERENCES Estudiante(id_estudiante),
    FOREIGN KEY (id_curso) REFERENCES Curso(id_curso)
);

-- Creación de índices
CREATE INDEX idx_estudiante_rut ON Estudiante(rut);
CREATE INDEX idx_profesor_rut ON Profesor(rut);
CREATE INDEX idx_curso_nombre ON Curso(nombre_curso);
CREATE INDEX idx_estudiante_curso_est ON Estudiante_Curso(id_estudiante);
CREATE INDEX idx_estudiante_curso_cur ON Estudiante_Curso(id_curso);
